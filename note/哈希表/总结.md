# 哈希表｜总结

关键：用空间换时间，利用哈希或计数结构快速查询和去重。

## 常见模式

- 计数/频次型
  - 使用 `unordered_map` / `vector`（字符只有 26/128/256 时）记录频次
  - 典型题：字母异位词（242）

- 去重/集合型
  - 使用 `unordered_set` 判重或判断元素是否存在
  - 典型题：两个数组的交集（349）

- 补数/映射型
  - 记录已遍历元素及其索引，用于快速查找补数
  - 典型题：两数之和（1）

- 循环检测型
  - 使用哈希集合记录中间状态，检测是否进入循环（快乐数等）

## 复杂度
- 常数级查询：O(1) 平均情况下（unordered_map/set）
- 时间：常见为 O(n)；一些构建型思路可能退化到 O(n^2)
- 空间：O(n)

## 易错点
- 不注意映射/集合大小导致不必要的空间开销
- 忽视哈希冲突/退化（极端情况下）
- 题目返回顺序或去重要求没有处理好
