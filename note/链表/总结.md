# 链表｜总结

关键：虚拟头节点统一处理、指针操作顺序、边界特判。

## 模板要点

- 虚拟头节点技巧
  - 创建 `dummy_head->next = head`，统一头节点的增删操作
  - 返回时 `return dummy_head->next`
  - 适用：删除节点、插入节点、交换节点

- 链表遍历删除
  - 状态：`temp` 指向待删节点的前驱
  - 操作：`temp->next = temp->next->next`
  - 易错：只有不删除时才移动 `temp`，否则跳过连续目标

- 链表反转
  - 迭代法：三指针 `prev, curr, next`，逐个翻转
  - 递归法：递归到尾部，回溯时 `head->next->next = head`
  - 易错：原头节点的 `next` 最后要置 `nullptr`

- 快慢指针
  - 找中点：快指针走两步，慢指针走一步
  - 找倒数第N个：快指针先走N步，然后同步走
  - 判环：快慢相遇则有环

- 环形链表入口
  - 哈希法：记录访问过的节点，重复即入口
  - 数学法：相遇后，头指针和相遇点同步走，再次相遇即入口

- 链表相交
  - 栈法：两链表入栈，从尾部开始比较
  - 双指针法：遍历完切换到另一链表，相遇点即交点

## 复杂度
- 遍历类：时间 O(n)，空间 O(1)
- 哈希/栈辅助：时间 O(n)，空间 O(n)

## 易错点
- 空链表特判 `if(!head) return nullptr`
- 删除节点前先保存，避免内存泄漏
- 双向链表需同时维护 `prev` 和 `next`
- 指针操作顺序：先保存、再断开、最后连接
